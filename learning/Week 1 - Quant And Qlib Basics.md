# Week 1 - Quant And Qlib Basics

## 学习目标

本周建立量化交易的基本框架。先不讨论代码实现，只理解一个策略从想法到验证要经历哪些环节。

完成后你应该能说清楚：

- 量化交易不是预测明天涨跌，而是系统性地验证市场规律。
- 一个策略必须包含市场假设、数据证据、交易规则和风险边界。
- Qlib 的作用是帮你把研究流程标准化，而不是替你发明赚钱逻辑。

## 核心问题

量化策略本质上是在回答：

> 市场中是否存在一种可重复、可解释、扣除成本后仍然有效的价格行为？

如果答案是“可能存在”，你需要继续问：

- 谁在提供这笔收益？
- 这个机会为什么没有被市场完全消除？
- 它依赖什么市场环境？
- 它会在哪些情况下失效？

## 策略收益从哪里来

常见策略来源可以先分成几类：

- 趋势：上涨的资产继续上涨，下跌的资产继续下跌。
- 反转：短期过度涨跌后回归均值。
- 价值：便宜资产长期跑赢昂贵资产。
- 质量：财务质量高、盈利稳定的公司长期更稳。
- 成长：高增长公司带来估值或盈利扩张。
- 波动：利用波动率、风险补偿或市场恐慌。
- 流动性：承担流动性不足带来的风险溢价。

机器学习模型只是帮助你组合信号。它不能替代策略假设。

## Qlib 在学习中的角色

Qlib 给你一条完整研究管线：

`数据 -> 信号 -> 模型 -> 排名 -> 持仓 -> 回测 -> 记录`

第一周你只需要知道每一段的含义：

- 数据：价格、成交量、股票池和交易日历。
- 信号：用历史数据构造出的预测线索。
- 模型：把多个信号组合成股票打分。
- 策略：根据打分决定买什么、卖什么、买多少。
- 回测：用历史市场检验这套规则。
- 复盘：判断结果是否可信，哪里可能过拟合。

## 本周实验

运行 LightGBM 示例，只把它当成一条样板策略：

```bash
source .venv/bin/activate
qrun examples/benchmarks/LightGBM/workflow_config_lightgbm_Alpha158.yaml
```

你不需要理解每个参数，只观察三类输出：

- 预测分数：模型认为哪些股票更有吸引力。
- IC / Rank IC：预测分数和未来收益是否同向。
- 回测指标：组合是否跑赢基准，回撤和成本是否可接受。

- 解释为什么它需要股票池、基准和交易成本。
- 记录你看到的 IC、超额收益、最大回撤。
- 在 [[Qlib Learning Log]] 写下：这个结果为什么还不能直接用于实盘。

## 常见误区

- 只看年化收益，不看回撤和成本。
- 把模型准确率当成策略质量。
- 忽略样本外验证。
- 不问数据是否有幸存者偏差或未来函数。
