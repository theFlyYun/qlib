# Quant Learning Roadmap

这是一条更完整的量化交易学习路线。Qlib 是练习平台，不是学习边界。

主线顺序是：

`市场常识 -> 数据源 -> 收益与风险 -> 因子信号 -> 策略规则 -> 回测验证 -> 组合风控 -> 机器学习 -> 实盘约束`

## 阶段 1：市场与交易常识

目标：先理解交易发生在哪里，以及策略到底要战胜什么。

要学：

- 股票、指数、ETF、期货、基金的基本差异。
- A 股和美股的交易规则差异。
- 开盘价、收盘价、成交量、成交额、停牌、涨跌停。
- 基准指数是什么，为什么不能只看绝对收益。
- 手续费、印花税、滑点、冲击成本。

结合 Qlib：

- 跑通 LightGBM 示例，观察 benchmark return 和 excess return。
- 先不研究模型，先理解“策略是否跑赢基准”。

阶段输出：

- 写一页笔记：一个股票多头策略必须说明哪些交易假设。

## 阶段 2：数据源与数据质量

目标：知道数据从哪里来，以及哪些数据问题会让策略结果失真。

要学：

- 行情数据：OHLCV、复权价、成交额、换手率。
- 财务数据：利润表、资产负债表、现金流、估值指标。
- 指数与行业数据：成分股、权重、行业分类。
- 另类数据：新闻、公告、资金流、宏观数据。
- 数据偏差：幸存者偏差、未来函数、财报延迟、缺失值、退市处理。

结合 Qlib：

- 使用当前简版 CN 数据学习流程。
- 认识 `Alpha158` 是基于价格和成交量的信号集合。
- 暂时不要把示例数据当成实盘数据。

阶段输出：

- 写一张“数据源验收表”：覆盖范围、频率、复权、退市、延迟、缺失、可复现性。

## 阶段 3：收益、风险与评价指标

目标：能看懂策略表现，不被漂亮收益曲线骗。

要学：

- 简单收益、对数收益、累计收益。
- 波动率、最大回撤、夏普比率、信息比率。
- Alpha、Beta、超额收益、跟踪误差。
- 收益分布、极端风险、尾部风险。
- 换手率和成本后收益。

结合 Qlib：

- 重点看回测输出里的 annualized return、information ratio、max drawdown。
- 比较 cost 前后的 excess return。

阶段输出：

- 用一段话解释：为什么“年化高”不等于“策略好”。

## 阶段 4：因子与信号

目标：理解因子不是公式堆砌，而是市场假设的量化表达。

要学：

- 动量：强者恒强。
- 反转：过度反应后的均值回归。
- 价值：便宜资产的风险溢价。
- 质量：盈利稳定和财务健康。
- 成长：盈利或收入扩张。
- 低波动：风险调整收益。
- 流动性：承担流动性风险的补偿。

结合 Qlib：

- 把 `Alpha158` 当作价格/成交量因子集合来理解。
- 关注 IC 和 Rank IC，而不是先追求复杂模型。

阶段输出：

- 选 3 个因子，分别写出：假设、数据、失效场景。

## 阶段 5：策略规则

目标：把信号变成可交易规则。

要学：

- 股票池如何定义。
- 买入规则、卖出规则、调仓周期。
- 持仓数量、单票权重、行业约束。
- Top-K、多空、市场中性、增强指数。
- 交易成本如何影响换手策略。

结合 Qlib：

- 学习 `TopkDropoutStrategy` 的思想：持有高分股票，限制每次替换数量。
- 调整 `topk` 和 `n_drop`，观察收益、回撤和成本变化。

阶段输出：

- 写一份策略卡片：信号、买入、卖出、调仓、成本、风险。

## 阶段 6：回测与验证

目标：学会怀疑回测结果。

要学：

- 样本内、样本外、滚动验证。
- 过拟合、数据泄露、未来函数。
- 参数调优陷阱。
- Walk-forward 验证。
- 金融机器学习里的 purged cross-validation 思想。

结合 Qlib：

- 把训练、验证、测试区间分开看。
- 不要反复用测试集调策略。
- 每次实验都记录假设和结果。

阶段输出：

- 给当前 LightGBM 示例写一份“回测可信度检查”。

## 阶段 7：组合构建与风控

目标：理解策略收益来自信号，生存来自风控。

要学：

- 等权、按分数加权、风险平价、优化器。
- 行业暴露、风格暴露、集中度。
- 止损、仓位控制、最大回撤控制。
- 风险模型和协方差矩阵。
- 主动管理中的信息比率和独立下注数。

结合 Qlib：

- 从 Top-K 组合开始，不急着做复杂优化。
- 再看 `examples/portfolio/` 的增强指数案例。

阶段输出：

- 写一份风控清单：单票、行业、换手、回撤、流动性、容量。

## 阶段 8：机器学习在量化中的位置

目标：知道机器学习能做什么，不能做什么。

要学：

- 标签设计：预测未来收益、方向、分位数。
- 特征选择和特征稳定性。
- 树模型、线性模型、神经网络的取舍。
- 模型解释、过拟合控制、样本外稳定性。
- Meta-labeling、组合多个信号。

结合 Qlib：

- LightGBM 是合适的第一模型。
- 深度模型先放后面，等你能解释数据和回测后再碰。

阶段输出：

- 写一段判断：当前问题是否真的需要机器学习。

## 阶段 9：实盘约束

目标：知道研究结果到真实交易之间差了什么。

要学：

- 数据延迟和数据修正。
- 下单、成交、滑点和撤单。
- 容量：策略规模变大后收益是否消失。
- 监控：收益漂移、信号失效、数据异常。
- 复盘：什么时候停用策略。

结合 Qlib：

- Qlib 更适合研究和回测，不要把它等同于完整交易系统。
- 实盘前需要独立的数据、执行、监控和风控系统。

阶段输出：

- 写一份“从回测到实盘还缺什么”的清单。

## 推荐节奏

- 第 1-2 周：阶段 1-2，市场和数据。
- 第 3-4 周：阶段 3-4，收益风险和因子。
- 第 5-6 周：阶段 5-6，策略和回测。
- 第 7-8 周：阶段 7，组合和风控。
- 第 9-10 周：阶段 8，机器学习。
- 第 11-12 周：阶段 9，实盘约束和复盘。
