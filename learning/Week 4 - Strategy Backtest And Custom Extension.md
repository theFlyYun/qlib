# Week 4 - Strategy Backtest And Custom Extension

## 学习目标

本周理解策略如何落地。信号只是排序观点，策略才决定买什么、卖什么、持有多久、承担多少风险。

完成后你应该能说清楚：

- 策略参数如何影响收益、换手和成本。
- 回测不是证明策略赚钱，而是排除明显不靠谱的想法。
- 风控不是附加项，而是策略的一部分。
- 一个策略从研究到实盘还需要经过哪些检查。

## 从信号到策略

一个常见股票多头策略可以这样理解：

1. 每天或每周给股票打分。
2. 选择分数最高的一组股票。
3. 卖出分数下降或不再入选的股票。
4. 控制持仓数量、调仓频率和交易成本。
5. 和基准指数比较超额收益。

Qlib 示例里的 `TopkDropoutStrategy` 可以简单理解为：

- 持有排名靠前的 `topk` 只股票。
- 每次调仓替换其中 `n_drop` 只。
- 通过限制替换数量降低换手和成本。

## 回测要看什么

不要只看收益。至少看这些维度：

- 年化收益：策略长期平均赚多少。
- 超额收益：相对基准多赚多少。
- 最大回撤：最糟糕时亏多少。
- 波动率：收益是否稳定。
- 信息比率：单位波动带来的超额收益。
- 换手率：交易频率是否过高。
- 交易成本：成本后收益是否还成立。

如果一个策略只在不计成本时赚钱，通常不可用。

## 风险边界

策略要提前定义失效条件：

- 市场风格切换后是否失效？
- 极端行情下回撤是否可承受？
- 持仓是否过度集中？
- 是否依赖无法稳定获得的数据？
- 是否对交易成本和滑点过于敏感？
- 是否只在某个特殊时间段有效？

没有失效条件的策略，很难复盘。

## 本周实验

复制一份配置做参数实验：

```bash
cp examples/benchmarks/LightGBM/workflow_config_lightgbm_Alpha158.yaml /tmp/qlib_topk_experiment.yaml
```

调整 `topk` 和 `n_drop` 后运行：

```bash
qrun /tmp/qlib_topk_experiment.yaml --experiment_name topk_experiment
```

观察重点：

- `topk` 变大后，收益和回撤是否更平滑？
- `n_drop` 变大后，收益是否提高，成本是否也提高？
- 扣成本前后结果差异有多大？

## 常见误区

- 把回测收益当成未来收益。
- 忽略成本和滑点。
- 只看一个市场阶段。
- 没有基准，也就无法判断是否真的有超额收益。
